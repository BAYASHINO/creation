1: PCのコーディングとの違い

[ スマホコーディングの特徴 ]

  • ・画像の使用は必要最低限にし、cssで実装できるところはcssを使うことで、極力ページサイズを小さくする。
  • ・ 縦位置（ポートレイト）と横位置（ランドスケープ）の両方を考慮する。
  • ・画像のファイルサイズ=1画像2MBという制限がある（一応）
  • ・retina対応させるので、デザインデータを2倍の大きさで制作する必要がある。

（今回はCanvasやSVGは使用していないので、そのあたりは割愛します。）

[ viewportの設定 ]

＊ width=device-width

200～10000pxの範囲で設定できる。
windowサイズに合わせて可変したいので、device-widthを指定する。
デフォルト値は980pxなので、指定しておかないとPCサイトのような表示になってしまいます。

＊ initial-scale=1

最初にページを表示したときのズーム率。
100%＝1.0として指定します。

＊ maximum-scale=1

最大ズーム率。100%＝1.0として指定します。
デフォルト値は1.6

＊ user-scalable="0"

拡大・縮小を有効にするかどうか。デフォルト値はオンになっているので、
無効にしたい場合は0かnoを指定する。

3: 注意点

[ 画像サイズ ]

画像を書き出す際、数値が奇数だとエッジがぼけたりすることがあるので、サイズは偶数にしておく方が無難。
（写真などだけに限らず、アイコンや見出しの画像なども同様。）

[ retina対応 ]

2倍のデータを見ていると大きく見えますが、実際の表示はそれの1/4になるので、コーディング完了後ブラウザで見ると、想定していたより小さくなることがあります。
デザイン段階でページ全体のjpgを書き出して1/4の大きさにしてみるなど、ある程度見えを確認しながら作ったほうが実装後の修正が少なくスムーズです。

[ 破線の処理 ]

破線を画像にするとぼけやすいので、cssで表示させることが多い。
その際、デザインの複雑なところに破線が使われていると実装が大変なので、可能であれば薄い実線を使うなどで回避した方が無難。PCほど細かくスライスを切りすぎると、実装が大変なことがある。

[ クラス名／ID名 ]

クラス名やIDの付け方、セクショニングをPCとスマホである程度合わせておくと、別の人がさわっても修正するときわかりやすい。

[scrollイベント ]

スマホでは、スクロール中は下記のような処理が止まってしまいます。

  • ＊ js…setTimeout、setInterval
  • ＊ css…transition、animation

また、jsのscrollイベントはスクロールが終わったタイミングでしか発生しないので、touchstart、touchmove、touchendなどのイベントも活用する。

[ bodyの中の子要素などを、別個でスクロールさせる場合 ]

iPhone、android共に、OSのバージョンによってスクロールの挙動が異なる。

  • ＊ 比較的新しいOS…-webkit-overflow-scrolling: touch;で1本指で子要素のスクロール可能。
    （バグが多めなのでiOS7、Android4より前のOSでは難しいかも知れません）
  • ＊ 幅広く対応させるには、iScrollを( http://cubiq.org/iscroll-4 )

=>とにかく動かない場合はjsのライブラリを使う！

[ UI設計時の注意点 ]

iOS7から、画面下部をタップすると、ツールバーが表示される仕様になりました。
そのため、ページ下部にベタ付きのフッターメニューなどを設けると、ボタンをタップしてもツールバーが表示されてしまい、うまくクリックできなかったりします。
デザイン段階で、そういったOSの仕様も考慮できると快適なUI設計ができそうです。

[ フォームのチェックボックス ]

iOS5ではlabelをクリックしてもcheckboxにチェックが入らないので、JSで解決してください。

4: デバッグ

スマホのコーディングをする際、使える便利なデバッグツールをまとめました。

[ chromeのDevelop Tools ]

  • ・user agentを切り替えることで、各OSの見えを確認できる。
  • ・OSの解像度以外のサイズも、数値を入力すれば任意のサイズで表示を切り替えてくれる。

※Develop Toolsの画面を閉じてリロードするとPC表示に戻ってしまうので気をつける。

[ Safariの開発メニュー ]

  • ・safariの開発メニューでもuser agentの切り替えができる。
  • ・実機をPCに接続し、実機のsafariの詳細設定でwebインスペクタをオンにすると、実機での表示を確認しながらソースを確認できる。

＊ iPhoneのSafariをPCに繋いで、インスペクタで調査できる

http://c-brains.jp/blog/wsg/12/09/28-093305.php

[ XcodeのiOSシミュレータ ]

XcodeのiOSシミュレータ + Safariのインスペクタで実機に近いの表示を確認しながらデバッグできる。
